Cabal Package Description Editor

Building projects in Eclipse relies on the Haskell Cabal, and to allow you to easily view and modify your project's Cabal file, there is an editor that supports Cabal's package description files.

Note that saving changes to the Cabal file will trigger a rebuild of the project.

Package

In the Package you can change the Cabal properties that affect the entire project. This includes descriptive properties such as project name, version, author, maintainer, license, description, homepage; and also properties that affect the build, such as minimal Cabal version.

Data files

Some packages need to access data files during its execution. The way Cabal has to deal with them can be configured in the Cabal editor:

  • Choose the files that will be available,
  • This will create a new Paths_<package> module which tells the paths to the files in the installed system,
  • Remember to include this module in the Other section for each component that will need access to the data files.

Components

Cabal deals with three kinds of components, that is, objects created from the source code. Those kinds are libraries, executables and test suites, and can be configured thorugh their respective tab in the Cabal editor.

In general, there are four configuration sections for each component:

  • Dependencies: which include information of other packages needed to build the project.
    You can add new dependencies clicking the + button. This will open a new window with the list of packages currently installed in your system.
    If you want, you can add a version requirement using the Version column in the table.
  • Modules: which tell the modules to gather to create the component. Each module can be in three different states: unselected; with the first column selected, that will expose the module (when editing a library) or will use it as entry point (when editing an executable or test suite); or with the second column selected, which will say that the module is a companion module.
  • Source folders: allow you to tell where the project files are located. When creating a new project, a src folder will be created, and it's recommended to place your files in that folder.
  • Compiler options: allow you to set flags or enable certain language extensions. Currently, this section only applies to GHC.

Each project can have at most one library. You can decide whether or not to build one in the Library tab.

On the other hand, the Executables and Test suites tabs allow to have as many components as you want for that type. On the left of the window you will find the list of components that you already have in your project. You can add more clicking the + button above the list.

For test suites, you can also select the type of test you are creating. Currently ony executable tests that use Test-framework can be run withing EclipseFP.

Source

The source tab provides and editor with syntax coloring, code folding, and makes also some templates available that let you easily create a new Cabal package description.

The editor is by default registered for the file extension .cabal. To trigger a template, hit Control+SPACE at any (non-comment) position in such a file. For some entries (e.g. the license-file entry, you can also hold the Control key and point with the mouse on the value - the entry is then changed into a link that takes you directly to the referenced file (if it exists) when you click on it.
