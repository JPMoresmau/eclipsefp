Haskell Editor

The Haskell source editor provides syntax coloring, Code Assist (context sensitive code completion), bracket matching and other helpful functionality to make coding in Haskell more convenient. If HLint is installed in your system, suggestions for improving your code will be shown too.

The editor supports both the conventional and the literal style of source code. Which style is used is determined from the file extension of the source file (.hs and .lhs for conventional and literate style resp.).

Code Assist is triggered by pressing CTRL-Space when typing. Code assist is context sensitive and can show you:

  • Functions, types and constructors in normal code
  • Types in type signatures
  • Modules in import statement
  • Pragma in LANGUAGE directives

The Outline View shows the overall structure of the Haskell source file in the currently active editor.

Certain regions of the editor can be folded into one line and unfolded with the small '-' and '+' buttons in the left editor ruler.

The Haskell worksheet views allows you to enter expressions evaluated when your file is changed. This is like GHCi except your expressions are preserved over save/reload of your file. The recalculation is done automatically. Several output formats like simple text or HTML are supported.

Configuration

Many of the features of the generic text editor can be configured in the Preferences at General > Editors > Text Editors. Some Haskell specific settings can be found under Haskell > Editor

Key shortcuts

Beside the standard text editing functionality (like Copy/Cut/Paste, Undo and Redo etc.) there are the following Haskell-specific shortcuts:

Shift + Ctrl + C Toggle comment (add or remove -- to the start of the selected lines)
Shift + Ctrl + P Add pragram comment ({-# ... #-} )
Shift + Ctrl + D F Add an Haddock comment before the current element (the element Follows the documention)
Shift + Ctrl + D P Add an Haddock comment after the current element
Shift + Ctrl + D B Add an Haddock block comment before the current element (the element Follows the documention)
Shift + Alt + R Rename the current element (if your project builds successfully, all instances of the element should be renamed everywhere they're used)
