Debugging

This plug-in supports debugging of Haskell modules using GHCi. That means that the features in this section are available only if the module is run within a GHCi session. To do this, right-click the module, and select Debug As > GHCi session.

You can add breakpoints in your code double-clicking in the left margin of the source code editor. A small blue circle will appear in the selected line. You can delete the breakpoint double-clicking one of those circles. Note that currently EclipseFP does not allow to set breakpoints in particular expressions, only in entire lines.

Once you start running functions in your module, it is recommended to change to the Debug perspective by going to the main menu and selecting Window > Open Perspective > Debug. This gives you access to the debugging tools.

When a breakpoint is reached, the expression being evaluated is shown with a different background in the code editor. You can step or continue using the buttons available in the Debug view.

You can also inspect the value of the variables in the Variables view. As withing GHCi, variables that aren't yet evaluated are shown as _. Typing something in the variable value field causes evaluation of the variable, which may modify the performance or behavior of your program.

You can also add Expressions in the Expressions view. You can create new expressions by typing the full code, and you can also select expressions in your code, right click and choose Watch. If you only want to see the value of an expression at one point in time you can just select the expression in the code, right click and choose Display. A popup will appear with the current value.

GHCi does not have the concept of stack traces (which are not always relevant in a lazy language). However it can keep the history of expressions it has evaluated. EclipseFP translates this history in stack frames in the Debug view. Just launch your function with :trace to get reliable history information (For example, if you want to launch your main function in GHCi, type in :trace main);

Debugging an executable

EclipseFP does not support direct debugging of executables. However, you can simulate it following these steps:

  • Right-click your main module and select debugging within GHCi,
  • If you havne't done it yet, add breakpoints to your code,
  • In the GHCi console, write main and press the Enter key. This will execute the main function, which is the entry point of your application.
